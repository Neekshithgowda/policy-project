1.DDL (Data Definition Language)
------------------------------
>CREATE: This command is used to create the database objects such as tables , relations , procedures, triggers, view etc...

SYNTAX:
----------
CREATE TABLE table_name
(
column_name_1 DATATYPE CONSTRAINT,
column_name_2 DATATYPE CONSTRAINT,
:
column_name_n DATATYPE CONSTRAINT,
);

CREATE TABLE ACCOUNT
(
ACCNO BIGINT PRIMARY KEY,
ACC_HOLDER_NAME VARCHAR(30) NOT NULL,
PHONE BIGINT UNIQUE NOT NULL CHECK(LENGTH(PHONE)=10),
PINCODE INT
);

CREATE TABLE BRANCH
(
BRANCHID INT PRIMARY KEY,
B_NAME VARCHAR(30)  UNIQUE,
PINCODE INT
);

CREATE TABLE LOCATION
(
PINCODE INT PRIMARY KEY,
AREA VARCHAR(20) UNIQUE,
CITY VARCHAR(20),
STATE VARCHAR(20) DEFAULT 'KARNATAKA'
);

*ALIAS:
======
It is a alternative name given to the columns/resultant table obtained by dql queries.

SYANTX:
SELECT column_name_1 AS alias_1,
       column_name_2  alias_2,
       column_name_3 " alias_3"
FROM TABLE NAME;

>We can pass alias name either by using AS keyword or "double quotes"
>With or without using As keyword , we can pass alias name
> we can pass spcl char or spaces by using the "double quotes"

FUNCTIONS:
==================
FUNCTIONS ARE THE BLOCK OF CODE WHICH IS UESD FOR THE PERFORM A SPECIFIC TASK.


TYPES OF FUNCTIONS:
>PRE DEFINED ( IN-BUILT FUNCTIONS)
>USER DEFINED (PL/SQL)

PRE DEFINED FUNCTIONS:
===========================
>Aggregate function
>Character function
>Number function
>Date function
>Window function

1.AGGREGATE FUNCTIONS:
-----------------------
IT TAKE A N NUMBER OF INPUT AND GENERATES A SINGLE OUTPUT

TYPES OF AGGREGATE FUNCTION:
========================

MAX(): This function is used to obtain the MAXMIMU value from the given column.
syntax: MAX (column_name);

MIN(): This function is used to obtain the MINIMUM value from the given column.
syntax: MIN (column_name);

SUM(): This function is used to obtain the TOTAL value from the given column.
syntax: SUM (column_name);

AVG(): This function is used to obtain the AVERAGE value from the given column.
syntax: AVG (column_name);

COUNT(): This function is used to obtain the NUMBER OF ROWS from the given column.
syntax: COUNT (*/column_name);

NOTE: ONLY COUNT(*) TAKES * AS A ARGUMENT.


CHARACTERSTICS OF AGGREGATE FUNCTION:
*AGGREGATE FUNCTIONS EXECUTE GROUP BY GROPU.
*WE CANNOT PASS ANY OTHER COLUMN ALONG WITH AGGREGATE FUNCTIONS
*WE CANNOT NEST AGGREGATE FUNCTIONS
*WE CANNOT PASS MULTIPLE COLUMNS INSIDE AGGREGATE FUNCTIONS
*WE CANNOT PASS AGGREGATE FUNBCTIONS INSIDE WHERE CLAUSE.
*AGGREGATE FUNCTIONS IGNORE NULL VALUES.


GROUP BY
> WE CAN PASS GROUP BY EXPRESSIONS ALONG WITH AGGREGATE FUNCTIONS.

NUMBER FUNCTIONS:
=====================
ABS(): THIS FUNCTION IS USED TO CONVERT THE NEGATIVE INTO POSITIVE NUMBERS.
SYNTAX:SELECT ABS(-20);===20
ABS(M)
 
MOD(): THIS FUNCTION IS USED TO OBTAIN THE REMAINDER OF THE GIVEN TWO NUMBERS.
SYNTAX:SELECT MOD(4,2);
MOD(M,N)
SELECT MOD(4,2);===0


ROUND(): THIS FUNCTION IS USED TO ROUND OFF THE GIVEN FLOATING VALUES.
SYNTAX:SELECT ROUND(4.56);
EX:ROUND(m,n);
ROUND(51276.33479);==========================51276
ROUND(51276.33479,2);=============================51276.33
ROUND(51276.33479,3);
ROUND(51276.33479,-1);
ROUND(51276.33479,-2);

CEIL(): THIS FUNCTION IS USED TO OBTAIN THE NEXT INTEGER VALUE FROM THE GIVEN DECIMAL NUMBER.


FLOOR(): THIS FUNCTION IS USED TO OBTAIN THE CURRENT INTEGER VALUE FROM THE GIVEN DECIMAL NUMBER.
ROUND DOWN TO THE LOWEST AND HIGHEST PRICES

POW(): THIS FUNCTION IS USED TO OBTAIN THE POWER OF THE GIVEN NUMBER .IT TAKES THE INTEGER VALUE AND THE DEGREE VALUE AS A ARGUMENT.
EX:SELECT POW(3,4);

RANK() AND DENSE_RANK()
===============================

THESE FUNCTIONS ARE USED TO ASSIGN THE RANKS FOR EVERY ROW PRESENT IN THE TABLE BASED ON THE WINDOW CONDITION.

RANK(): RANK() FUNCTIONS ASSGINS THE RANK OF THE CURRENT ROW NUMBER.
SYNATX: RANK() OVER(ORDER BY/PARTITION BY column_Name ASC/DESC;

DENSE RANK(): DENSE_RANK() FUNCTION ASSGINS THE RANK FOLLOWING THE SEQUENCE OF THE RANKS.
SYNATX: DENSE_RANK() OVER(ORDER BY/PARTITION BY column_Name ASC/DESC;

 **********************************************************************************
OVER(): THIS CLUASE IS USED TO PASS THE WINDOW CONDITIONS . WE CAN PASS OVER() IN TWO TYPES .

1.ORDER BY(): IT IS USED TO ARRANGE THE RECORDS EITHER IN ASCENDING OR DESECNDING OPRDER.
SYNTAX:window_function OVER (ORDER BY column_name ASC/DESC);

2.PARTITION BY(): IT IS USED TO DIVIDE THE RECORDS BASED ON CERTAIN COLUMN OR EXPRESSION.
SYNTAX:window_function OVER (ORDER BY column_name ASC/DESC);

SUB QUERY:
==============
A QUERY IINSIDE ANOTHER QUERY IS KNOWN AS SUB QUERY.

WORKING PRINCIPLE OF SUB QUERY:
--------------------------------
* SUB QUERY CONSISTING OF INNER QUERY AN OUTER QUERY
* ALWAYS INNER QUERY WILL EXECUTE FIRST AND GENERATES THE OUTPUT
* THE OUTPUT QUERY OF THE INNER QUERY WILL BE GIVEN AS INPUT TO THE OUTER QUERY
* HERE, THE OUTPUT QUERY DEPENDS ON THE OUTPUT OF THE INNER QUERY:
 
1.RULES TO WRITE SUB QUERY:
____________________________
1. WHERE SAL>(SELECT FNAME - WRONG 
         DECIMAL != VARCHAR
   WHERE SAL>(SELECT COMM - RIGHT 
         DECIMAL = DECIMAL
IN SUBQUERY , THE DATATYPER MUST MACTH BETEEN INNER QUERY AND OUTER QUERY.


2. WHERE SAL=(SELECT  SAL,COMM - WRONG 
   WHERE SAL>(SELECT SAL - RIGHT 
         IN SUBQUERY , WE CAN PASS ONLY SINGLE COLUMN INSIDE THE INNER QUERY.


WHEN OR WHERE WE CAN GO FOR SUB QUERY:
-----------------------------------
CASE 1 : WHENEVER WE COME ACROSS UNKNOWN VALUES, WE GO FOR SUB QUERY.
CASE 2 : WHENEVER THE DATA IS IN ONE TABLE AND THE CONDITION WE PASS IS IN ANOTHER TABLE, WE GO FOR SUBQUERY.

===================================================================================
DRAWBACKS OF SUBQUERY
================================
> IT IS DIFFCULT TO CONNECT MULTIPLE TABLES.
> WE CANNOT ABLE TO GET THE DATA FROM TWO DIFFERENT TABLES AT ONCE.

************************

JOINS:
******
JOINS ARE USED TO RETRIEVE THE DATA FROM MULTIPLE TABLES SIMULTANEUSLY.

TYPES OF JOINS
**************
> CARTESIAN JOIN/CROSS JOIN(DEPRECATED)
> INNER JOIN
> SELF JOIN
> OUTER JOIN
    -> LEFT OUTER JOIN
    -> RIGHT OTER JOIN
    -> FULL OUTER JOIN
>NATURAL JOIN

**************************

CARTESIAN JOINS/CROSS JOINS
============================
IN CROSS  JOIN,THE RECORDS OF TABLE 1 WILL BE MERGED WITH RECORDS OF TABLES2.


WHENEVER WE PEROFORM CROSS JOINS,
-----------------------------------------
NO OF COLUMNS GENERATED: T1+T2
NO ROWS GENERATED :T1*T2


SYNTAX:
***
SELECT COLUMN_NAME
FROM TABLE_NAME1 CROSS JOIN TABLE_NAME 2;


DRAWBACK
********
> WE GET MORE NUMBER OF ERROR WHEN COMPARED TO VAILD RECORDS.






NOTE:
======
INNER JOIN ARE USED TO OBTAIN ONLY THE MATCHED RECORDS

SYSNTAX:
=-------
SELECT COLUMN_NAME 
FROM table_name_1 INNER JOIN table_name2
ON table_name1.coloumn_name=table_name2.column_name;

OUTER JOINS
==============
OUTER JOINS ARE USED TO OBTAIN THE MATCHED AND UNMATCHED RECORDS.

IN OUTER JOINS WE HAVE,
-> LEFT OUTER JOIN
-> RIGHT OUTER JOIN
-> FULL OUTER JOIN


LEFT OUTER JOINS
==================
LEFT OUTER JOINS ARE USED TO OBTAIN THE MATCHED AND THE UNMATCHED RECORDS FROM THE LEFT TABLE.

SYNTAX: 
********
SELECT COLUMN_NAME
FROM TABLE_NAME T1 LEFT OUTER JOIN TABLE_NAME T2
ON T1.COLUMN_NAME=T2.COLUMN_NAME;

RIGHT OUTER JOINS
=================
RIGHT OUTER JOINS ARE USED TO OBTAIN THE MATCHED AND THE UNMATCHED RECORDS FROM RIGHT TABLE.

SYNTAX:
*******
SELECT COLUMN_NAME
FROM TABLE_NAME T1 RIGHT OUTER JOIN TABLE_NAME T2
ON T1.COLUMN_NAME=T2.COLUMN_NAME;

FULL OUTER JOINS
================
FULL OUTER JOINS ARE USED TO OBTAIN THE MATCHED AND THE UNMATCHED RECORDS FROM BOTH THE TABLE.
> IT IS THE COMBINATION  OF THE BOTH LEFT OUTER JOIS AD RIGHT  OUTER JOINS.

SYNTAX:
******
SELECT COLUMN_NAME
FROM TABLE_NAME T1 LEFT OUTER JOIN TABLE_NAME T2
ON T1.COLUMN_NAME=T2.COLUMN_NAME
UNION
SELECT COLUMN_NAME
FROM TABLE_NAME T1 RIGHT OUTER JOIN TABLE_NAME T2
ON T1.COLUMN_NAME=T2.COLUMN_NAME;
